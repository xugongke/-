## 1. 系统简介

### 1.1 系统概述

本系统是一套**太阳能光伏热水器智能管理平台**，用于将光伏板发出的直流电能智能分配给整栋楼各户的热水器使用。系统通过**电力线载波通信（PLC）**技术，在已有的电力线上实现主机与各户从机之间的数据传输，无需额外布设通信线路。

### 1.2 核心功能

| 功能 | 说明 |
|------|------|
| **智能负载调度** | 根据光伏板实时发电功率，自动调整接入的热水器数量，最大化太阳能利用率 |
| **用户用电量计量** | 精确记录每户的日、月、年累积用电量，为收费提供依据 |
| **LCD本地显示** | 4.3寸彩色触摸屏，本地查看设备状态和用电数据 |
| **上位机远程管理** | 通过电脑软件远程管理所有设备，搜索、入网、查看数据 |
| **4G无线通信** | 内置4G模块，支持远程数据上传和天气信息获取 |
| **以太网通信** | 内置W5500以太网芯片，支持与上位机有线连接 |
| **实时时钟** | 内置RX8025T高精度时钟芯片，自动同步网络时间 |
| **数据存储** | 所有用户数据存储在SD卡中，断电不丢失 |

### 1.3 与传统方案的对比优势

| 对比项 | 传统MPPT控制柜 | 本系统 |
|--------|---------------|--------|
| 电能转换效率 | 需经过DC-DC变换器，效率约95% | 直接供电，无变换器损耗 |
| 综合利用率 | 受限于变换器功率 | 智能调度负载，利用率更高 |
| 通信方式 | 通常无通信 | 电力线载波，无需额外布线 |
| 本地显示 | 无或仅有指示灯 | 4.3寸彩色LCD触摸屏 |
| 用户计量 | 需额外安装电表 | 内置精确计量功能 |
| 成本 | 高（大功率DC-DC变换器） | 低（仅需继电器） |

### 2.2 设备清单

| 设备 | 数量 | 说明 |
|------|------|------|
| 主控制器 | 1台 | 含LCD屏、通信模块，安装在楼栋机房或配电箱内 |
| 从机模块 | 每户1个 | 含ES1642通信模块+继电器，安装在用户热水器旁 |
| 光伏板 | 按需配置 | 安装在楼顶，直流输出连接到主控制器 |
| 网线 | 1根 | 连接主控制器与上位机电脑 |
| SD卡 | 1张 | 插入主控制器卡槽，用于存储用户数据 |
| SIM卡 | 1张 | 插入主控制器4G模块，用于网络通信 |

### 2.3 硬件安装步骤

#### 2.3.1 主控制器安装

1. 将主控制器固定在楼栋机房或配电箱内
2. 连接光伏板直流输出线到主控制器的**光伏输入端子**
3. 连接主控制器的**电力线输出**到楼栋的电力线（同时为从机供电和通信）
4. 用网线连接主控制器的**以太网口**到上位机电脑（或交换机）
5. 将SD卡插入主控制器的**SD卡插槽**
6. 将SIM卡插入主控制器的**4G模块卡槽**
7. 接通主控制器的**220V交流电源**（用于控制系统本身供电）

#### 2.3.2 从机模块安装

1. 将从机模块（ES1642+继电器）安装在每户的热水器旁
2. 将从机模块的**电力线输入**连接到该户的电力线
3. 将从机模块的**继电器输出**连接到热水器的**直流加热元件**
4. 确保从机模块的地址拨码开关处于**默认状态**（搜索后由主机统一分配地址）

> **注意**：从机模块通过电力线与主机通信，**无需额外布设通信线**。只需确保从机与主机连接在同一相电力线上，通信距离最远可达**500米**。

## 3. 系统上电与启动

### 3.1 上电顺序

请按以下顺序给系统上电：

1. **确认所有硬件连接正确**
2. **接通主控制器电源** → LCD屏亮起，显示启动画面
3. **等待约10秒** → 系统自动完成以下初始化：
   - SD卡加载
   - 4G模块联网
   - 以太网获取IP地址
   - RTC时钟同步网络时间
4. **LCD屏自动跳转到主页面** → 系统就绪

## 5. 上位机软件操作指南

### 5.1 连接主机

1. 确保上位机电脑与主控制器在同一局域网内（通过网线或交换机连接）
2. 打开上位机管理软件
3. 在LCD屏主页面查看当前主机的**IP地址**和**端口号**
4. 在上位机软件中输入IP地址和端口号，点击**"连接"**
5. 连接成功后，软件显示"已连接"状态

### 5.2 搜索设备

连接成功后，需要进行设备搜索，将所有从机注册到系统中：

1. 点击上位机软件的**"搜索设备"**按钮
2. 主机通过电力线向所有从机广播搜索命令
3. 从机收到搜索命令后自动回复，上报自己的MAC地址
4. 搜索完成后，LCD屏和上位机都会显示搜索到的设备列表
5. 新搜索到的设备初始状态为**"未入网"**

> **提示**：搜索过程可能需要10~30秒，取决于从机数量和通信距离。

### 5.3 设备入网（绑定地址）

搜索到的设备需要设置通信地址并入网后才能正常使用：

1. 在上位机软件中选择一个**未入网**的设备
2. 设置该设备的**通信地址**（楼栋号、单元号、房间号），例如：
   - 楼栋：1，单元：1，房间：0101 → 地址为 `01:01:01:01:00:00`
3. 点击**"绑定"**或**"入网"**按钮
4. 主机通过电力线向该从机发送：
   - 新的通信地址
   - 统一的网络口令（PSK）
5. 从机收到后自动入网，LCD屏和上位机状态更新为**"已入网"**
6. 系统自动为该用户创建数据文件，开始记录用电量

### 5.4 查看用户数据

设备入网后，可以随时查看每户的用电数据：

- **方式一**：在上位机软件中点击对应用户查看
- **方式二**：在LCD屏上进入设备列表，点击用户进入详情页

### 5.5 操作流程图

```
上位机连接主机 → 搜索设备 → 显示设备列表（未入网）
                              │
                              ↓
                    逐一设置通信地址（楼栋/单元/房号）
                              │
                              ↓
                    点击"入网" → 从机加入网络
                              │
                              ↓
                    状态变为"已入网" → 开始记录用电量
                              │
                              ↓
                    查看日/月/年累积用电量 → 按用量收费
```

## 6. 日常工作流程

### 6.1 系统运行阶段

设备全部入网后，系统进入**自动运行阶段**，无需人工干预：

1. **光伏发电**：光伏板将太阳能转化为直流电能
2. **智能调度**：主控制器实时监测光伏输出功率，根据发电量自动决定接通多少台热水器
3. **负载匹配**：
   - 当阳光充足、发电量大时 → 同时接通多台热水器
   - 当阳光变弱、发电量小时 → 自动减少接通的热水器数量
   - 优先加热水温最低的热水器，确保公平性
4. **数据记录**：系统持续记录每户的用电量，自动统计日/月/年数据
5. **数据存储**：所有数据保存在SD卡中，断电不丢失

### 6.2 每日自动任务

| 时间 | 自动执行的操作 |
|------|---------------|
| 凌晨 0:00 | 日用电量自动归零，开始新一天计量 |
| 每月1号 0:00 | 月用电量自动归零，开始新月计量 |
| 每年1月1日 0:00 | 年用电量自动归零，开始新年计量 |
| 每15分钟 | 自动获取最新天气信息 |
| 持续运行 | 智能调度热水器通断，最大化太阳能利用 |
| 持续运行 | 通过电力线与所有从机保持通信 |

### 6.3 收费管理

运营方可根据累积用电量向用户收费：

1. **按月收费**：每月初查看上月各用户的"月用电量"
2. **收费标准**：根据当地电价政策制定（如 0.5元/kWh）
3. **数据导出**：通过上位机软件导出用户用电数据
4. **账单生成**：根据月用电量 × 单价 = 应收金额

## 7. 用电量数据说明与收费标准

### 7.1 数据层次说明

```
总用电量（total_energy）     ← 永不归零，记录设备生命周期总消耗
├── 年用电量（annual_energy） ← 每年1月1日归零，年度统计
│   ├── 月用电量（monthly_energy） ← 每月1号归零，月度收费依据
│   │   └── 日用电量（daily_energy）   ← 每天0点归零，日常监控
```

### 7.2 收费示例

假设电价标准为 **0.55元/kWh**：

| 用户 | 本月用电量 | 应收金额 |
|------|-----------|---------|
| 1栋1单元0101 | 156.80 kWh | 86.24 元 |
| 1栋1单元0102 | 132.50 kWh | 72.88 元 |
| 1栋2单元0201 | 178.20 kWh | 98.01 元 |
| ... | ... | ... |

### 7.3 数据安全

- 所有用电数据存储在主控制器的SD卡中
- 每个用户有独立的数据文件（以"楼栋_单元_房间.bin"命名）
- 数据文件包含魔数校验，防止数据被篡改
- **断电后数据不丢失**，下次上电自动加载

## 8. 常见问题与故障排除

### 8.1 LCD屏相关问题

| 问题 | 可能原因 | 解决方法 |
|------|---------|---------|
| LCD屏不亮 | 电源未接通 | 检查主控制器电源线 |
| 4G信号灰色 | SIM卡未插入或信号弱 | 检查SIM卡；将4G天线移至信号好的位置 |
| 时间不正确 | 首次上电未同步网络时间 | 等待几分钟自动同步；检查4G网络 |
| IP地址显示0.0.0.0 | 网线未连接 | 检查网线连接 |

### 8.2 通信相关问题

| 问题 | 可能原因 | 解决方法 |
|------|---------|---------|
| 搜索不到设备 | 从机未上电或电力线不通 | 检查从机电源；确保同一相电力线 |
| 入网超时 | 通信距离过远或干扰大 | 减少通信距离；检查电力线质量 |
| 已入网设备掉线 | 从机断电或模块故障 | 检查从机供电；更换ES1642模块 |
| 上位机无法连接 | IP地址或端口错误 | 核对LCD屏显示的IP和端口 |

### 8.3 数据相关问题

| 问题 | 可能原因 | 解决方法 |
|------|---------|---------|
| 用电量显示"-- kWh" | 数据文件损坏或版本不匹配 | 删除SD卡中该用户的bin文件，重新入网 |
| SD卡写入失败 | SD卡损坏或已满 | 更换SD卡；清理旧日志文件 |
| 时间显示20xx年00月 | RTC电池电量不足 | 更换RX8025T模块的纽扣电池 |

## 10. 注意事项

### 10.1 安全注意事项

?? **警告**：本系统涉及强电操作，安装和维护必须由**专业电工**进行！

1. 安装前必须**断开总电源**
2. 确保所有接线牢固，无裸露导线
3. 主控制器必须**可靠接地**
4. 光伏板直流输出具有较高的电压，操作时注意安全
5. 禁止在潮湿环境中安装和使用

### 10.2 使用注意事项

1. **SD卡**：请使用正品SD卡（推荐8GB~32GB，Class10以上），首次使用前格式化为FAT32
2. **SIM卡**：请确保SIM卡已开通数据业务，且有足够流量
3. **网线**：请使用标准网线（Cat5e及以上），长度不超过100米
4. **电力线**：确保所有从机与主机在同一相电力线上，否则无法通信
5. **定期检查**：建议每季度检查一次接线是否松动，SD卡是否正常

### 10.3 维护保养

1. 保持主控制器通风良好，避免阳光直射
2. 定期清洁LCD触摸屏（用软布轻擦）
3. 每半年检查一次SD卡剩余空间
4. 每年更换一次RTC纽扣电池（CR2032）

## 附录B：通信地址编码规则

通信地址为6字节，编码规则如下：

| 字节 | 内容 | 示例 |
|------|------|------|
| 字节0 | 楼栋号（1~255） | 0x01 = 1栋 |
| 字节1 | 单元号（1~255） | 0x01 = 1单元 |
| 字节2 | 房间号高字节 | 0x01 |
| 字节3 | 房间号低字节 | 0x01 → 房间号 = 0x0101 = 257（即0101室） |
| 字节4 | 预留（0x00） | - |
| 字节5 | 预留（0x00） | - |

**示例**：1栋2单元0305室 → 地址为 `01:02:03:05:00:00`

## 1. 产品概述

本产品是一款**基于负载侧智能投切的太阳能光伏热水器控制系统**，用于将楼顶光伏板发出的直流电能高效分配给整栋楼各户的热水器使用。

系统采用**电力线载波通信（PLC）**技术，通过已有的电力线实现主机与各户从机之间的数据传输和控制，**无需额外布设通信线路**，安装简便，改造成本低。

### 核心特点

- **无DC-DC变换器**：省去了传统MPPT所需的昂贵变换器，效率更高、成本更低
- **智能负载投切**：根据光伏实时发电功率自动调整接入的热水器数量
- **电力线载波通信**：通过电力线同时传输电能和通信数据，无需额外布线
- **LCD液晶显示**：配备4.3寸彩色LCD屏，支持按键操作，本地查看设备状态
- **精确计量收费**：记录每户日/月/年累积用电量，为收费提供数据依据
- **上位机管理**：通过电脑软件远程管理所有设备

## 2. 工作原理

### 2.1 光伏发电基本原理

太阳能光伏板的输出特性具有非线性特征，其输出功率会随着光照强度、温度等因素变化。在特定条件下，光伏板存在一个**最大功率点（MPP）**，在该点处光伏板输出的电功率最大。

```
光伏板P-V特性曲线（功率-电压关系）

  P(功率)↑
        │        ╱╲
        │       ╱  ╲ ← 最大功率点(MPP)
        │      ╱    ╲       此处功率最大
        │     ╱      ╲
        │    ╱        ╲
        │   ╱          ╲
        │  ╱            ╲
        │ ╱              ╲
        │╱                ╲
        └────────────────────→ V(电压)
         0        Vmp      Voc
```

### 2.2 传统MPPT方案（DC-DC变换器）

传统方案使用**DC-DC变换器**（如Buck/Boost电路）来实现最大功率点跟踪：

```
┌──────┐     ┌─────────────┐     ┌──────────┐
│光伏板 │────→│ DC-DC变换器  │────→│ 负载(热水器)│
│      │     │ (MPPT控制器) │     │          │
└──────┘     └─────────────┘     └──────────┘
                   ↑
            通过调节占空比
            改变等效负载阻抗
```

**工作方式**：DC-DC变换器通过快速调节开关管的占空比，改变从光伏板侧看到的等效负载阻抗，使之始终等于最佳负载阻抗，从而实现最大功率点跟踪。

**传统方案的缺陷**：
- DC-DC变换器本身存在约5%的功率损耗
- 大功率DC-DC变换器成本高昂
- 变换器产生电磁干扰，影响电力线通信
- 需要散热，增加系统复杂度
- 变换器是易损件，维护成本高

### 2.3 本系统方案（负载侧智能投切）

本系统采用完全不同的思路：**不改变光伏板的工作点，而是改变负载大小来匹配光伏板的最大功率输出**。

```
┌──────┐                    ┌──────────┐
│光伏板 │────────────────────│ 从机1     │ ← 继电器ON/OFF
│      │   直流母线          │ 热水器1   │
│      │                    ├──────────┤
│      │                    │ 从机2     │ ← 继电器ON/OFF
│      │                    │ 热水器2   │
│      │                    ├──────────┤
│      │                    │ 从机3     │ ← 继电器ON/OFF
│      │                    │ 热水器3   │
│      │                    ├──────────┤
│      │                    │  ...      │
│      │                    ├──────────┤
│      │                    │ 从机N     │ ← 继电器ON/OFF
└──────┘                    └──────────┘
     ↑                            ↑
  主控制器通过电力线载波     每个从机有一个
  控制各从机继电器通断       固定阻值的加热器

  接入N个加热器并联 → 总负载 = R_加热器 / N
  通过调节N的大小 → 使总负载匹配光伏板最佳工作点
```

**核心原理**：

假设每个热水器的加热电阻为R，当同时接入N个热水器并联时，总负载电阻为 R/N。

| 接入数量N | 总负载电阻 | 效果 |
|-----------|-----------|------|
| N太小（如1个） | R/1 = R（很大） | 光伏电压偏高、电流偏小，功率未充分利用 |
| N太大（如50个） | R/50（很小） | 光伏电压被拉低、电流过大，功率也偏低 |
| **N合适** | **R/N = R_optimal** | **匹配最大功率点，输出功率最大 ?** |

### 2.4 智能投切算法（扰动观察法）

系统采用**改进型扰动观察法（P&O）**实时跟踪最大功率点：

```
算法流程：

        ┌─────────────────────┐
        │ 1. ADC采样光伏电压V  │
        │    和电流I            │
        │    计算功率P=V×I      │
        └──────────┬──────────┘
                   │
                   ▼
        ┌─────────────────────┐
        │ 2. 比较当前功率P与   │
        │    上次功率P_last     │
        └──────────┬──────────┘
                   │
           ┌───────┴───────┐
           │               │
      P > P_last       P < P_last
      功率增大 ?        功率减小 ?
           │               │
           ▼               ▼
    继续当前方向      反向调整
    (N+1 或 N-1)     (从+1变为-1)
           │               │
           └───────┬───────┘
                   │
                   ▼
        ┌─────────────────────┐
        │ 3. 更新P_last = P    │
        │    等待系统稳定       │
        │    (约5~10秒)        │
        └──────────┬──────────┘
                   │
                   ▼
              返回步骤1
```

### 2.5 公平调度策略

系统在投切过程中确保所有用户公平使用太阳能：

1. **温度优先**：优先给水温最低的热水器通电加热
2. **轮询调度**：所有用户轮流获得加热机会，不会出现某户永远无法加热的情况
3. **目标温度控制**：达到目标温度的热水器自动退出调度队列，将机会让给其他用户
4. **用电量均衡**：系统记录各户累计用电量，长期保持各户之间的公平性

## 3. 与传统DC-DC MPPT方案的对比优势

### 3.1 效率对比

| 对比环节 | 传统DC-DC MPPT | 本系统（负载投切） |
|---------|---------------|------------------|
| 光伏跟踪精度 | 99%以上（连续调节） | 95%~98%（离散调节） |
| DC-DC变换器效率 | 约95%（5%能量损耗） | **不需要变换器，0损耗** |
| **综合效率** | 99% × 95% = **94.05%** | 96% × 100% = **96%** |
| 结论 | — | **综合效率更高** |

### 3.2 成本对比

| 成本项 | 传统DC-DC MPPT | 本系统（负载投切） |
|--------|---------------|------------------|
| DC-DC变换器 | 需要，大功率型号数千元 | **不需要** |
| MPPT控制器 | 需要专用芯片 | 不需要，软件算法实现 |
| 每户从机 | 仅需电表 | 仅需继电器+通信模块 |
| **系统总成本** | **高** | **低30%~50%** |

### 3.3 可靠性对比

| 可靠性指标 | 传统DC-DC MPPT | 本系统（负载投切） |
|----------|---------------|------------------|
| DC-DC变换器故障 | 常见故障点，整个系统停机 | 无此部件 |
| 继电器故障 | — | 固态继电器寿命极长 |
| 散热需求 | 大功率变换器需强制散热 | 无需额外散热 |
| 电磁干扰 | 开关变换器产生强干扰 | 无开关噪声 |
| 维护难度 | 需专业人员维修 | 模块化设计，更换简单 |

### 3.4 功能对比

| 功能 | 传统DC-DC MPPT | 本系统（负载投切） |
|------|---------------|------------------|
| 本地LCD显示 | 通常没有 | ? 4.3寸彩色LCD屏+按键操作 |
| 单户用电计量 | 需额外安装电表 | ? 内置精确计量 |
| 远程管理 | 通常不支持 | ? 上位机软件远程管理 |
| 通信能力 | 通常无 | ? 电力线载波通信 |
| 天气信息 | 无 | ? 4G联网获取实时天气 |
| 数据存储 | 无或有限 | ? SD卡永久存储 |

### 3.5 总结

```
传统方案：  光伏 → DC-DC变换器(损耗5%) → 负载     成本高、效率低
本系统：    光伏 → 智能投切(无损耗) → 多个负载     成本低、效率高
```

### 4.3 硬件连接步骤

#### 主控制器连接

1. 光伏板直流输出线 → 主控制器的**光伏输入端子**
2. 主控制器的**电力线输出** → 楼栋电力线
3. 网线 → 主控制器**以太网口** → 上位机电脑
4. SD卡插入**SD卡插槽**
5. SIM卡插入**4G模块卡槽**
6. 接通主控制器**220V交流电源**

#### 从机模块连接

1. 从机模块安装在每户热水器旁
2. 电力线输入 → 该户电力线
3. 继电器输出 → 热水器**直流加热元件**
4. 确保从机与主机在**同一相电力线**上

> **注意**：从机通过电力线通信，**无需额外布设通信线**。通信距离最远**500米**。

### 5.7 上位机软件操作

#### 5.7.1 连接主机

1. 确保上位机电脑与主控制器在同一局域网内
2. 打开上位机管理软件
3. 在LCD屏主页面查看当前**IP地址**和**端口号**
4. 在上位机输入IP地址和端口号，点击**"连接"**

#### 5.7.2 搜索设备

2. 主机通过电力线广播搜索命令
3. 从机自动回复，上报MAC地址

4. 主机通过电力线发送地址和口令

## 6. 用电量数据与收费说明

### 6.1 数据层次

系统为每户记录四个层次的累积用电量：

```
总用电量（total_energy）      ← 永不归零，设备生命周期总消耗
└── 年用电量（annual_energy）  ← 每年1月1日归零，年度统计
    └── 月用电量（monthly_energy） ← 每月1号归零，月度收费依据
        └── 日用电量（daily_energy）   ← 每天0点归零，日常监控
```

| 数据项 | 显示位置 | 归零周期 | 用途 |
|--------|---------|---------|------|
| 日用电量 | LCD屏详情页 | 每天0点 | 日常监控 |
| 月用电量 | LCD屏详情页 | 每月1号 | **月度收费依据** |
| 年用电量 | LCD屏详情页 | 每年1月1日 | 年度统计 |
| 总用电量 | 内部记录 | 永不归零 | 设备档案 |

### 6.2 收费方式

运营方根据**月累积用电量**向用户收费：

```
应收金额 = 月用电量(kWh) × 单价(元/kWh)
```

**示例**（假设电价 0.55元/kWh）：

| 用户 | 月用电量 | 应收金额 |
|------|---------|---------|
| 1栋1单元0101 | 156.80 kWh | 86.24 元 |
| 1栋1单元0102 | 132.50 kWh | 72.88 元 |
| 1栋2单元0201 | 178.20 kWh | 98.01 元 |

### 6.3 数据安全

- 每户独立数据文件，存储在SD卡中
- 数据文件含魔数校验，防篡改
- **断电不丢失**，上电自动加载

## 7. 常见问题与故障排除

### 7.1 LCD屏相关

| 问题 | 原因 | 解决方法 |
|------|------|---------|
| LCD屏不亮 | 电源未接通 | 检查电源线 |
| LCD背光自动关闭 | 正常节能功能 | 按任意按键唤醒 |
| 4G信号灰色 | SIM卡未插或信号弱 | 检查SIM卡；调整天线位置 |
| 时间不正确 | 首次上电未同步 | 等待几分钟自动同步 |
| IP地址显示0.0.0.0 | 网线未连接 | 检查网线 |

### 7.2 通信相关

| 问题 | 原因 | 解决方法 |
|------|------|---------|
| 搜索不到设备 | 从机未上电/电力线不通 | 检查从机电源；确保同一相电力线 |
| 入网超时 | 距离过远或干扰大 | 减少距离；检查电力线质量 |
| 上位机无法连接 | IP/端口错误 | 核对LCD屏显示的IP和端口 |

### 7.3 数据相关

| 问题 | 原因 | 解决方法 |
|------|------|---------|
| 用电量显示"-- kWh" | 数据文件损坏 | 删除SD卡中该用户的bin文件，重新入网 |
| SD卡写入失败 | SD卡损坏或已满 | 更换SD卡 |

## 9. 注意事项

### 9.1 安全注意事项

?? **警告**：本系统涉及强电操作，安装和维护必须由**专业电工**进行！

1. 安装前必须**断开总电源**
2. 确保所有接线牢固，无裸露导线
3. 主控制器必须**可靠接地**
4. 禁止在潮湿环境中安装和使用

### 9.2 使用注意事项

1. **SD卡**：推荐8GB~32GB，Class10以上，FAT32格式
2. **SIM卡**：需开通数据业务
3. **网线**：Cat5e及以上，长度不超过100米
4. **电力线**：所有从机与主机必须在**同一相电力线**上
5. **定期检查**：每季度检查接线、SD卡

### 9.3 维护保养

1. 保持主控制器通风良好
2. 定期清洁LCD屏（软布轻擦）
3. 每年更换RTC纽扣电池（CR2032）
